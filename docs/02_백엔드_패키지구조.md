# 백엔드 패키지 구조 설계서 (FastAPI)

> **대상:** 개발팀
> **목적:** FastAPI 백엔드의 디렉터리/패키지 구조와 각 모듈의 책임을 정의해 개발 착수 기준을 맞춘다.
> **관련 문서:** [01_아키텍처_설계서.md](./01_아키텍처_설계서.md)
> **전제:** DB(벡터DB 등)는 **이미 로컬 도커에 떠 있는 컨테이너에 연결**한다. Dockerfile/compose는 현 단계에서 작성하지 않는다.

> ⚠️ **구현 현황(as-built):** 설계 초안(§2.1/§3.1)과 실제 구조(§2.2/§3.2)를 함께 싣는다. 상세는 [12_구현_아키텍처.md](./12_구현_아키텍처.md). 주요 차이:
> - **`mcp_servers/` 는 chatBot-sample 에서 제거됨** → 상위 `git/` 폴더의 **외부 독립 프로젝트**(`faq-*`, `general-*`, FastAPI+FastMCP).
> - `app/mcp/` 는 `client.py`(인프로세스) 대신 **`faq_client`·`domain_client`(원격 MCP)** 만. 도구 카탈로그는 정적 파일 없이 서버 `list_tools` 디스커버리(`scripts/index_tools.py`).
> - **`pizza-backend`(FastAPI+PostgreSQL) 신규** — general-pizza 가 정형/실시간 데이터를 이 백엔드 API 로 조회(§2.2).
> - `scripts/`: `init_db`·`seed_demo`·`index_tools`·`run_external_servers`.

---

## 1. 설계 원칙

1. **계층 분리** — API(입출력) / Service(비즈니스) / Infra(검색·LLM·DB)를 분리한다.
2. **아키텍처 컴포넌트 = 패키지** — 설계서(01)의 각 컴포넌트가 폴더로 1:1 대응되어, 문서를 보고 코드 위치를 바로 찾을 수 있게 한다.
3. **배포 단위 분리** — 메인 오케스트레이터(`app`)와 MCP 서버들(`mcp_servers`)은 독립 배포 단위이므로 최상위에서 분리한다.
4. **설정 외부화** — host/port, 테넌트 매핑 등은 코드에 하드코딩하지 않고 `.env`/`configs`로 분리한다.

| 프로젝트 | 책임 | 비고 |
|---|---|---|

## 4. 의존성 방향 (단방향 유지)

```
api → services → orchestration → {retrieval, llm, memory, mcp, tenancy, autocomplete}
                                          ↓
                                     core (config / logging / exceptions)
```

> 안쪽(인프라)은 바깥(api)을 알지 못한다. 결합도를 낮추고 교체·테스트를 쉽게 한다.

---

## 5. 핵심 포인트 (개발팀 합의용)

1. **`app`(오케스트레이터) vs `mcp_servers`(MCP) 최상위 분리**
   - 두 영역은 독립 배포 단위. MCP 서버는 **LLM이 없는 "도구 제공자"**임이 구조에서 드러난다.
2. **`faq_template`은 단일 코드**
   - 업체별로 코드를 복제하지 않는다. 이 템플릿 하나를 `config.yaml`만 바꿔 N개 인스턴스로 배포한다(§6 원칙).
3. **`api`는 얇게, 로직은 `services`/`orchestration`에**
   - 엔드포인트는 입출력 변환만. 실제 흐름은 LangGraph로. 테스트 용이.
4. **설정 외부화**
   - DB host/port, 테넌트 매핑은 `.env`/`configs`로. 코드 하드코딩 금지.

## 6. DB 연결 방침 (기존 로컬 컨테이너 사용)

DB는 이미 로컬 도커에 떠 있으므로, **앱은 클라이언트로 접속만** 한다. 연결 정보는 `.env` → `app/core/config.py`(Pydantic Settings)에서 읽어 `app/retrieval/vector_store.py` 등에서 사용한다.

```ini
# .env.example — 기존 로컬 도커 컨테이너 접속 정보
QDRANT_HOST=localhost
QDRANT_PORT=6333
```
